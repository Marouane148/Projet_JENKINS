# GUIDE DE CONFIGURATION JENKINS

## ÉTAPE 1 : CONFIGURATION INITIALE

1. Accéder à http://localhost:8080
2. Entrer le mot de passe initial (obtenu via la commande docker exec)
3. Cliquer sur "Install suggested plugins" ou sélectionner les plugins manuellement
4. Créer le premier utilisateur administrateur (données non requises pour test local)
5. Valider les configurations par défaut

---

## ÉTAPE 2 : CONFIGURER JDK 17 ET MAVEN 3

### 2.1 Configuration de JDK17

1. Aller à **Manage Jenkins** (en haut à gauche)
2. Cliquer sur **Tools and Plugins** → **Tools** (ou **Global Tool Configuration** selon la version)
3. Chercher la section **JDK**
4. Cliquer sur **Add JDK**
5. Configurer comme suit:
   - Nom: `JDK17`
   - Chemin: `/usr/lib/jvm/java-17-openjdk-amd64` (déjà présent dans le conteneur)
   - ☑ Cocher "Install automatically" (optionnel)
6. Cliquer sur **Save**

### 2.2 Configuration de Maven3

1. Dans la même page **Global Tool Configuration**
2. Chercher la section **Maven**
3. Cliquer sur **Add Maven**
4. Configurer comme suit:
   - Nom: `Maven3`
   - Chemin: `/usr/share/maven` (déjà présent dans le conteneur)
   - ☑ Cocher "Install automatically" (optionnel)
5. Cliquer sur **Save**

---

## ÉTAPE 3 : CRÉER UN JOB PIPELINE

### 3.1 Créer un nouveau job

1. Cliquer sur **+ New Item** (ou **New Job**)
2. Entrer le nom: `Projet-JENKINS-Pipeline`
3. Sélectionner **Pipeline**
4. Cliquer sur **OK**

### 3.2 Configurer le Pipeline

Dans l'écran de configuration du job:

#### Section: General
- Description: `Build et test du projet Maven ICDE848`
- ☑ GitHub project
  - URL du projet: `https://github.com/Marouane148/Projet_JENKINS`

#### Section: Pipeline
Sélectionner: **Pipeline script from SCM**

- **SCM**: Git
  - Repositories
    - Repository URL: `https://github.com/Marouane148/Projet_JENKINS.git`
    - Credentials: (none - public repo, ou ajouter si private)
  - Branches to build:
    - Branch Specifier: `*/main`
  - Script Path: `Jenkinsfile`

5. Cliquer sur **Save**

### 3.3 Créer/Mettre à jour le Jenkinsfile

Si votre Jenkinsfile n'existe pas, créez-le avec ce contenu:

```groovy
pipeline {
    agent any

    options {
        buildDiscarder(logRotator(numToKeepStr: '10'))
        timeout(time: 1, unit: 'HOURS')
    }

    environment {
        JAVA_HOME = '/usr/lib/jvm/java-17-openjdk-amd64'
        MAVEN_HOME = '/usr/share/maven'
        PATH = "${MAVEN_HOME}/bin:${JAVA_HOME}/bin:${PATH}"
    }

    stages {
        stage('Checkout') {
            steps {
                echo 'Récupération du code depuis GitHub...'
                checkout scm
            }
        }

        stage('Build') {
            steps {
                echo 'Compilation avec Maven...'
                sh 'mvn clean compile'
            }
        }

        stage('Test') {
            steps {
                echo 'Exécution des tests...'
                sh 'mvn test'
            }
        }

        stage('Code Quality') {
            steps {
                echo 'Analyse Checkstyle...'
                sh 'mvn checkstyle:check'
            }
        }

        stage('Package') {
            steps {
                echo 'Création du JAR...'
                sh 'mvn package -DskipTests'
            }
        }

        stage('Integration Tests') {
            steps {
                echo 'Tests d\'intégration...'
                sh 'mvn verify'
            }
        }
    }

    post {
        always {
            junit 'target/surefire-reports/*.xml'
            recordCoverage(
                tools: [[parser: 'JACOCO', pattern: 'target/site/jacoco/jacoco.xml']],
                name: 'JaCoCo'
            )
        }
        success {
            echo 'Build réussi ✓'
        }
        failure {
            echo 'Build échoué ✗'
        }
    }
}
```

### 3.4 Tester le Pipeline

1. Cliquer sur **Build Now**
2. Observer l'exécution en temps réel dans **Console Output**
3. Vérifier les résultats des tests et couverture de code

## ÉTAPES SUPPLÉMENTAIRES (OPTIONNEL)

### Activer l'authentification GitHub (personal access token)

Si votre repository est privé:

1. Aller à **Manage Jenkins** → **Credentials** → **System** → **Global credentials**
2. Cliquer sur **+ Add Credentials**
3. Kind: `Username with password` ou `GitHub App`
4. Username: votre GitHub username ou `Marouane148`
5. Password: Token GitHub avec permissions `repo` et `admin:repo_hook`
6. Dans la configuration du job Pipeline, sélectionner ces credentials

### Configurer l'intégration webhook GitHub

Pour que Jenkins soit notifié automatiquement lors d'une push:

1. Aller sur GitHub: https://github.com/Marouane148/Projet_JENKINS/settings/hooks
2. Cliquer sur **Add webhook**
3. Payload URL: `http://votre-ip:8080/github-webhook/`
4. Content type: `application/json`
5. Events: `Push events`, `Pull requests`
6. Cliquer sur **Add webhook**

## DÉPANNAGE

### Jenkins ne démarre pas
- Vérifier les logs: `docker-compose logs jenkins`
- Vérifier que le port 8080 est libre: `netstat -ano | findstr :8080`
- Vérifier que Docker Desktop fonctionne

### Could not resolve java version 17
- Vérifier l'installation Java dans le conteneur:
  `docker exec jenkins-master java -version`

### Maven non trouvé
- Vérifier que Maven est installé:
  `docker exec jenkins-master mvn -version`

### Plugin Git ne fonctionne pas
- Vérifier que le plugin est installé dans Jenkins
- Redémarrer Jenkins: `docker-compose restart jenkins`
